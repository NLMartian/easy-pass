Easy Pass Privacy Policy

Last updated: June 10, 2026

Easy Pass is a static, end-to-end encrypted password manager. It runs in your browser and uses Dropbox only to store and synchronize one encrypted vault file named vault.enc in the app's Dropbox App Folder.

Information We Collect

We do not operate a backend server and we do not collect, receive, sell, or share your personal information. Easy Pass does not send your vault contents to any server controlled by us.

Information Stored Locally

  • Your Dropbox OAuth token is stored in your browser's local storage so the app can sync with Dropbox.
  • Your Dropbox app key setting may be stored in your browser if you override the bundled app key.
  • Your master password is never stored. The derived encryption key exists only in page memory while the vault is unlocked.

Information Stored in Dropbox

Password entries, custom fields, TOTP secrets, notes, tags, and password history are encrypted locally before upload. Dropbox receives only the encrypted vault.enc file and related file metadata needed to store and synchronize it.

Dropbox API Usage

Easy Pass requests Dropbox OAuth permission for files.content.read, files.content.write, and files.metadata.read. These permissions are used only to read, upload, and check metadata for vault.enc in the app's Dropbox App Folder.

Security

The vault is encrypted in the browser using a key derived from your master password with Argon2id and encrypted with AES-256-GCM. If you forget the master password, we cannot recover your vault.

Data Control and Deletion

  • You can remove the local Dropbox token from the app settings.
  • You can revoke Dropbox access from your Dropbox account settings.
  • You can delete the encrypted vault by deleting vault.enc from the app's Dropbox App Folder.

Contact

For questions about this policy, open an issue at github.com/NLMartian/easy-pass.
